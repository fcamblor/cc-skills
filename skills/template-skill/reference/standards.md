# Standards Reference

This file documents the standards and conventions your skill applies.

## Example: Code Quality Standards

- **Naming**: Clear, descriptive names following conventions
- **Documentation**: Inline comments for complex logic
- **Testing**: All public functions include test cases
- **Error Handling**: Graceful failure with meaningful messages
- **Performance**: O(n) or better for operations on large datasets

## Example: Documentation Standards

- Sentences end with periods
- Code blocks include language identifiers
- External links are provided with context
- Examples are complete and runnable

## Skill Development Standards

When building skills for this repository:

1. **Clarity**: Descriptions must be specific enough for Claude to discover the skill
2. **Focus**: Each skill handles one primary domain
3. **Examples**: Include practical, runnable examples
4. **Documentation**: Cover limitations explicitly
5. **Organization**: Use subdirectories (reference, templates, scripts)
